# 3. 贴图浮窗的优先级 + 统一 z-order 协调者

状态:已接受(2026-06-19)

## 背景

继 ADR-0002 给外部「窗口置顶」加了"按置顶时间严格排序"后,贴图浮窗(`PinWindow`,本程序自己的 Qt 窗口)也要可调优先级,且用户提出两条硬约束:

1. 贴图优先级用**离散等级**(右键菜单二级菜单选),定为 **1~5,1 最高、5 最低,默认 3**;跨级严格(高级恒压低级,点低级也不冒头),同级宽松(按"最后点击"排)。
2. **所有贴图浮窗整体高于所有外部置顶窗**——两个大层:贴图层在上、外部窗层在下。

矛盾点:外部窗的 z-order 由 `WindowPinner` 的定时器持续把它们压到 `HWND_TOPMOST` 顶;贴图浮窗带 `WindowStaysOnTopHint`,也在同一个 OS topmost band。两套若各自为政,`WindowPinner` 把外部窗压顶时会盖住贴图,违反约束 2。

## 决策

**让 `WindowPinner` 升格为全局唯一的 z-order 协调者**,一次 `_restack()` 排定整个 topmost band:

- **总顺序(从低到高依次 `SetWindowPos(HWND_TOPMOST, SWP_NOACTIVATE)`,后压者在上)**:
  外部置顶窗(按置顶时间升序) → 贴图浮窗(按 `priority` 5→1,同级按"激活序号"升序)。
  于是最高优先级、最近激活的贴图落在全局最上;所有贴图恒高于所有外部窗。
- **贴图注册**:`tray._create_pin` 创建 `PinWindow` 后 `register_pin(win)`,协调者接其 `activated`(点击→刷新激活序,同级置顶)、`priority_changed`(改优先级/切换始终置顶→重排)、`closed`(注销)三个信号。
- **触发重排**:① 贴图被点(`activated`)② 改优先级/切换置顶(`priority_changed`)③ 外部窗被点(`_enforce` 定时检测前台变化)④ 框选结束恢复隐藏的贴图(`tray` 调 `reassert`)。
- **同级宽松**靠"激活序号"实现:点哪张哪张序号最大→排本级最上;非置顶(关了"始终置顶")的贴图不参与重排。

## 否决的备选

1. **贴图各自维持 z-order(不引入协调者)**:`PinWindow` 自己 `raise_()`。但它无法知道外部窗的存在,保证不了"贴图恒高于外部窗",且多张贴图互相不知道彼此优先级。已否决。
2. **贴图优先级用时间序(同外部窗)**:与外部窗模型统一。但用户明确要"右键菜单里选档位",只能是离散等级,时间序无法从菜单"选"。已否决。

## 后果

- **好**:单一 `_restack` 是整个 band 层级的唯一真相源,跨子系统一致;贴图/外部窗的优先级规则各自表达又统一落地;复用了 ADR-0002 已有的 enforce 定时器与"前台变化才重排"的省力策略。
- **代价**:
  - `WindowPinner` 职责变重(名为"窗口置顶"实则管全局 z-order)——未来若拆分需小心两层的耦合。
  - 贴图用 `int(win.winId())` 取 HWND 交给 Win32 `SetWindowPos`,把 Qt 窗口混进 Win32 z-order 管理;若 Qt 在某些时机自行 `raise`,可能与协调者短暂打架(由 enforce/信号兜底纠正)。
  - 同 ADR-0002:持续重压有闪烁风险;重叠场景下点下层窗会被上层盖住(已知接受)。
- **约束**:`register_pin` 必须在 `win.show()` 之后调用,否则 `winId()` 尚未创建原生句柄。
