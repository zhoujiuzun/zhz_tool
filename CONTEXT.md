# 术语表 (CONTEXT.md)

> 本文件是项目的**领域术语词典**,只定义概念与边界,不含实现细节。
> 重构 / 加功能时如出现与此处冲突的用词,以本文件为准或先在此澄清。

## 处理阶段 (Pipeline Stages)

工具的主线是一条「取图 → 识别 →(可选)翻译 → 展示」的流水线,各阶段概念上不可混用。但注意:**翻译并不只是这条流水线的下游阶段**——它同时是一个可**单独调用**的功能,此时输入文字直接来自手动输入/剪贴板,不经过取图与识别(详见「翻译」条目的「入口」)。即:识别永远产出文字交给(可选的)翻译,但翻译不一定来自识别。

### 取图 (Capture)
获取一张待处理图片的动作。当前有两个来源:**截图框选**(屏幕框选)与**剪贴板监控**(自动检测剪贴板图片)。取图的产物始终是一张图片(PNG 字节),不含任何文字概念。

### 识别 (OCR / Recognition)
把一张图片转成**纯文字(原文)**的动作。由「**OCR 接口**」执行。原文是识别阶段的唯一产物。

### 翻译 (Translation)
把一段**待翻译文字**转成**译文**的动作。由「**翻译接口**」执行,与 OCR 接口是两套独立的接口池。
- **输入来源**:翻译的输入是「一段待翻译文字」,**不限于识别产物**。可来自三处:
  1. **识别产物**:OCR 识别出的原文(「从 OCR 进入」)。
  2. **手动输入/粘贴**:在译文窗的原文框中打字或粘贴。
  3. **剪贴板文字**:译文窗提供「翻译剪贴板」按钮,点击即抓取剪贴板中的**纯文字**进行翻译(显式触发,非自动预填)。注意此处抓的是**文字**,与「取图」里抓**图片**的剪贴板监控是两回事。
- **入口**:翻译有两个入口,二者最终都进入同一个**译文窗**:
  1. **从 OCR 进入**:识别后,原文带入翻译(由「自动翻译」开关或译文窗手动入口触发)。
  2. **单独调用**:从托盘直接打开译文窗,以手动输入/粘贴或「翻译剪贴板」按钮取得待翻译文字。
- 原文(待翻译文字)在翻译后**始终保留**,以支持原文/译文对照。
- **触发时机**(仅指「从 OCR 进入」这条路):由「自动翻译」开关(配置项,默认关闭)决定是否在识别后自动翻译;无论开关如何,译文窗始终提供手动翻译入口。即「可配置默认 + 手动覆盖」。
- **窗口置顶**:译文窗是否始终置顶**可切换**(用户诉求:用浏览器时把译文窗钉在上层方便随时翻)。
- **目标语言 (Target Language)**:译文要翻成的语言。
  - **源语言**:一律自动检测,不需用户指定。
  - **默认方向规则**:译文窗打开后的**初始**状态,第一次翻译即按此自动判定。
    - 源语言**非中文** → 翻成**中文**。
    - 源语言**为中文** → 翻成**英文**。
  - **粘性目标语言 (Sticky override)**:译文窗提供目标语言选择。一旦手动选定某个具体语言,即**完全盖过**方向规则——其后所有文字(**含中文**)都翻成该语言,直到改选别的语言。
    - **作用范围 = 当前译文窗的生命周期**:窗口关闭即失效;**下次打开译文窗回归默认方向规则**。
    - 不提供"切回自动"的入口:初始态本就是方向规则,手动选定后粘住即可。

## 接口 (Providers)

「接口」指一个具体的云服务端点。项目中有两类**互不通用**的接口:

### OCR 接口 (OCR Provider)
执行**识别**阶段的接口。图片 → 原文。如 Mistral、Gemini、Azure、百度、腾讯、讯飞。
- 注意:Gemini / Mistral 这类视觉大模型虽然*技术上*能一步识别并翻译,但在本项目中它们**只承担识别职责**,不承担翻译职责。识别与翻译刻意解耦,是为了保留原文做对照、并让翻译能作用于**所有** OCR 接口的结果(而非被绑死在 LLM 接口上),代价是多一次网络调用。这是架构基线。

### 翻译接口 (Translation Provider)
执行**翻译**阶段的接口。原文 → 译文。与 OCR 接口共享同一套「多接口 + 优先级 + 失败回退」的派发模式,但是**独立的接口池、独立的优先级、独立的配置**。

第一版内置五个翻译接口:
- **DeepL** — 译文质量主力,有免费额度。
- **Google 翻译** — 覆盖语言最广、质量稳。与作为 OCR 体系的 Gemini 同属 Google,但**翻译用的是另一套 API / 另一套计费**,是独立接口条目。
- **Gemini** — 复用用户已配的 Gemini key(prompt 方式翻译),零额外注册的兜底。注意:此处 Gemini 作为**翻译接口**与它作为 **OCR 接口**是两个独立条目,key 可共用但配置互不影响。
- **百度翻译** — 复用百度体系,易接入,有免费额度。与「百度 OCR」是不同的 API / 不同的接口条目。
- **有道智云** — 学习场景口碑之选,有免费额度。

## 译文窗 (Translation Window)

承载「原文 → 译文」的统一窗口,即流水线中的**展示**环节。**OCR 识别后展示**与**单独调用翻译**用的是**同一个窗**,不是两个。

- **结构**:上下分栏 —— 上为原文框,下为译文框,两框均可编辑。
- **初始原文来源**:窗口接收一个「初始原文」(可为空)。从 OCR 进入则预填识别结果;单独调用则为空,等待手动输入/粘贴,或点「翻译剪贴板」按钮取剪贴板文字。
- **操作**:翻译按钮、翻译剪贴板按钮、粘性目标语言选择(在译文区上方)、复制原文 / 复制译文两个按钮。
- **置顶**:是否始终置顶可切换(方便对照浏览器边看边翻)。
- 此窗即术语「翻译」条目中「入口」所说的两个入口共同进入的窗口。

## 贴图 (Pin / Sticky Image)

把一次**截图框选**的图片钉成一个**浮窗**(默认置顶,可切换),留在屏幕上,用于把不同页面的内容并排对照(学习场景)。

- 与「识别 → 翻译」流水线**完全独立、不联动**:贴图不识别、不翻译,浮窗上也不提供识别/翻译入口。它只是一张钉在屏幕上的图。
- 入口独立:与「截图识别」分属两个截图入口,框选前即决定本次是识别还是贴图。
- 浮窗 (Pinned Window):贴图产生的置顶图片窗口。交互细节:
  - **拖动**:按住浮窗任意位置可拖到屏幕任意位置摆放。
  - **多张**:可同时存在多个浮窗,用于并排对照不同页面。
  - **缩放**:鼠标点中浮窗后,**Ctrl + 滚轮**等比缩放。
  - **关闭**:右键菜单「关闭」,或选中后按 **Esc**。
  - **置顶层级**:浮窗是否始终置顶**可在右键菜单中切换**(右键菜单的「浮窗设置」),非永久写死。
  - **贴图优先级 (Pin Priority)**:多张置顶浮窗之间的上下层等级,**1~5 共五档,1 最高、5 最低,新建默认 3**,在浮窗右键菜单「优先度」二级菜单里选定。规则:**跨级严格**——高优先级浮窗恒压低优先级浮窗,点击低级浮窗也不会使其冒到高级之上;**同级宽松**——同一档的多张浮窗之间按"最后点击"排,点哪张哪张到本级最上。要让某张恒在上,把它调高一级即可。
  - **与外部「窗口置顶」的层级关系**:**所有贴图浮窗整体高于所有外部置顶窗**——即贴图层恒在外部窗层之上,任一贴图浮窗都压住任一外部置顶窗(见「窗口置顶」条)。
  - **透明度**:第一版不做。
- **与截图框选的边界**:发起「截图识别 / 截图贴图」框选时,已存在的浮窗**不得遮挡框选区域**(框选期间浮窗需临时下沉或不拦截鼠标),否则无法框选浮窗下方的内容。

## 动作序列 / 宏 (Action Sequence / Macro)

一段**有序的输入动作**,可被**回放**(按顺序执行)并可循环。这是「连点器」诉求扩展后的最终形态:不再只是"按间隔重复单一鼠标点击",而是"把**多种输入动作**编成一串按顺序重放"。与「取图 → 识别 → 翻译 → 展示」流水线及「贴图」**完全独立、不联动**:它只产生输入事件,不取图、不识别、不翻译。这是 app 从「OCR 工具」扩展为「桌面工具箱」后,其中一个**自成一体的工具**。

- **动作 (Action)**:序列中的一个输入事件。涵盖鼠标与键盘两类:
  - **鼠标**:左 / 右 / 中键、**侧键**(后退/前进键)的点击;双击;**滚轮滚动**;光标**移动**到某坐标;**拖拽**。
  - **键盘**:某个按键的敲击(或按下 / 抬起)。
  - **等待 (Delay)**:序列中两个动作之间的停顿,本身也是一种动作。
    - **随机延迟 (Random Delay)**:等待动作除固定时长外,可附加一段**随机抖动**——实际停顿 = 固定时长 + 0 到「随机上限」之间的随机量。用于让节奏不完全机械(拟人、防简单频率检测)。
- **回放 (Playback)**:按序列顺序依次执行各动作的动作。「回放中」是一个**持续状态**。
- **循环 (Loop)**:把整条序列重复执行 —— 一次 / 固定次数 / 无限(直到停止)。
- **启停 (Start / Stop)**:让回放开始或停止(同一个 toggle 动作)。**每条宏可配自己的回放热键**——按哪条宏的热键就启停哪条宏的回放;也可在配置页用按钮触发。录制的启停则由一个**全局**录制热键控制(空闲按下开始录制当前选中宏、再按停止)。
- **目标位置**:涉及位置的动作(点击 / 移动 / 拖拽)落在屏幕哪个坐标 —— 可**跟随当前光标**或**锁定指定坐标**。
  - **绝对坐标 (Absolute) vs 相对坐标 (Relative)**:锁定坐标分两种语义——**绝对**是屏幕固定像素点(原点为屏幕左上角);**相对**是以**回放时当前光标位置**为原点的偏移量(X/Y 可负)。相对坐标使同一条宏在不同起点复用同一段位移轨迹,不被绑死在固定屏幕点。

## 主题色 (Theme Color)

用户可在「设置 → 通用」自定义的**单一品牌色**,默认是 app logo 的浅蓝。它是全局视觉的**唯一可调输入**:界面其余配色(浅色背景、按钮悬停/按下态、选中高亮、边框、焦点描边等)都由它**自动派生**,用户不单独调这些。

- **作用边界**:主题色驱动「强调元素 + 由它派生的浅色背景」。**正文文字颜色不随之改变**(固定为深灰),以保证任何主题色下文字都清晰可读——这是「可定制」与「可用性」的硬边界。
- **风格基线**:整体为**清新风**——浅色背景(主题色的极淡衍生)+ 饱和主题色作强调,而非大面积铺满高饱和色。
- **生效范围**:改动**全局即时生效**,所有已打开与新开的窗口统一换色,无需重启。
- **默认与重置**:首次使用为 logo 浅蓝;提供「恢复默认」回到该色。除自由取色外,另提供若干**清新预设色**供一键选用。

## 窗口置顶 (Window Pin / Always-on-Top)

把**任意外部程序的窗口**(浏览器、PDF、聊天窗等)钉到屏幕最上层,方便边看边操作别的。是一个与 OCR/翻译/宏/贴图并列的**独立桌面小工具**,只操作窗口层级,不取图、不识别、不读内容。**可同时钉多个窗口**;多个置顶窗之间的上下层关系由「置顶顺序」决定(见下)。

- 与「贴图」的区别:贴图是把一张**静态截图**钉成浮窗(本工具自己的窗口);窗口置顶是把**别的程序活的窗口**提到顶层(那个窗口仍可正常交互)。两者都"钉在上层",但一个钉图片、一个钉真窗口。
- **触发方式**:看哪个窗口就让它在前台(点它),按**全局热键**(默认 `Ctrl+Alt+T`,可在通用里改/留空禁用)→ 把**当前前台窗口**钉到最上层;再按一次 → 取消(toggle)。**仅全局热键一个入口**——托盘菜单不再提供此项(菜单触发取窗不可靠,已移除)。
- **置顶顺序 (Pin Order)**:多个置顶窗之间的固定上下层关系,**按"置顶的先后时间"排定——后钉的压在先钉的之上**,且**严格维持**:点击一个较早置顶的窗,它**不会**因此冒到更晚置顶的窗之上(与普通窗口"点谁谁置顶"不同)。要够到被压住的较早置顶窗,需先移走或最小化压在它上面的窗。
- **提到最上**:把某个较早置顶的窗重新变为"最新"(从而升到所有置顶窗的最上层)的动作 = **对它重新置顶一次**(先取消再置顶)。这是改变「置顶顺序」的唯一方式——顺序不手动排,只随置顶时间走。
- **判定**:读窗口的 `WS_EX_TOPMOST` 真实状态翻转,不靠记录,鲁棒。
- **边界**:作用对象是当前前台窗口;无法置顶比本程序权限更高的窗口(如管理员窗,UIPI 限制)。本工具自己的窗口置顶另有其途径(译文窗置顶开关、贴图浮窗默认置顶),不走此工具。
- **与「贴图」的层级关系**:**所有贴图浮窗整体高于所有外部置顶窗**——故外部置顶窗永远在贴图浮窗之下,无论二者各自的优先级/置顶顺序如何(见「贴图 → 与外部窗口置顶的层级关系」)。

## 功能可见性 (Feature Visibility)

用户可在「设置 → 通用」按**功能模块**开关其可见性,把用不到的模块整体隐藏,避免入口与设置项过多显杂。隐藏一个模块 = 同时隐去它的**托盘菜单入口**与**设置窗对应 Tab**。

- **粒度 = 功能模块**(非单个按钮):如「OCR 识别」「翻译」「宏」「截图贴图」「窗口置顶」等,各为一个开关。
- **不可隐藏的红线**:托盘菜单的「设置」「退出」、设置窗的「通用」Tab **永远显示**。否则用户藏掉后将无法再进入设置把它开回来(死锁)。所有可见性开关都集中在「通用」,故任何隐藏都可逆。
- **隐藏 ≠ 卸载**:被隐藏的模块功能仍在(配置、引擎、热键照常),只是不在 UI 露出;重新勾选即恢复入口。

## 更新检查 (Update Check)

软件已在 GitHub 开源,**只检查是否有新版本并提示用户,不自动下载或替换**——下载、安装由用户手动到 GitHub 完成。这样不消耗任何自有服务器/CDN 流量。

- **判定依据**:本地 `version.__version__` 与 **GitHub 最新 Release 的 tag**(如 `v1.0.1`)按语义化版本比较;远端更高即视为有新版。
- **检查时机**:① 启动后延迟数秒**静默检查**一次——**仅在有新版时**弹托盘气泡(点击跳转 GitHub Release 页),无新版或网络失败都不打扰;② 设置窗「关于」页的「检查更新」按钮**手动检查**,无论结果如何都给出反馈。
- **边界**:不下载、不替换程序文件;网络失败保守处理(视为无更新,静默)。发版动作(建 Release、传新包)在 GitHub 侧人工完成,软件只读取、不写入。
